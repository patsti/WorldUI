General Info

This project is using Java/Spring boot for both the Rest-API project and the GUI. Animation is done with the Lottie-lib.

Project dependency descriptor

  • WorldUI

    • Java/spring boot is the development environment. Using Thymeleaf to ease GUI-creation.

    Workflow once game is started
    1. ./game-start, this path will generate a random seed, that is sent to the API to fetch a challenge
    2. ./game (redirected from /game-start), this function will send a request to the API to receive the challenge and visualize it
    3. ./gameredir (redirected when a guess is selected). If it's the first item then redirect to ./game, if it's the second selected item it will redirect to ./result, for validation
    4. ./result, another redirecting path. This will run the GameResponseObject.isCorrectGuess(). Then send the response into ./game-start which is then passed to ./game for visualization
  • WorldRestApi

    • Java/spring boot is the development environment. Using JPA to sync between Database and rest-api. The persist objects are identified by the naming, always ending with PO (ex. CityPO). This project also contains serializable versions of the persit objects (ex. City). These are used for external communication. The main object is called GameResponseObject. The purpose of this is to send a standalone quiz/challenge that can verify it self. once the challenge can be completed by a function called "isCorrectGuess". This will look at the guesses that has been put into the GameResponseObject and do a validation-check, returning true/false

  • Database

    • Postgres is used as main DB. The data that I've been using is the world.sql.gz. \n this consist of three tables, City, Country, CountryLanguage
